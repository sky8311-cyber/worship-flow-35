
목표: 이번 수정에서 **스트링 라이트 앵커링, 차량 레인 정렬/주행, 드럼 비율, 창문 밝기 통일**을 한 번에 확정 해결합니다.

1) `src/components/worship-studio/StudioSidePanel.tsx` — 스트링 라이트 “건물 최상단 연결” 고정
- 문제 원인: `RooftopStringLights`가 `style={{ bottom: '100%' }}`라서, 폴대 기준점이 건물 상단선이 아니라 루프탑 컨테이너 위로 떠 있음.
- 수정:
  - `RooftopStringLights`의 절대 위치를 건물 상단선 기준으로 재고정 (`bottom: 0` 기준으로 변경).
  - 폴대 하단 Y(`poleBottomY`)를 SVG 하단과 정확히 맞추고, 폴은 위로만 뻗도록 유지.
  - 스트링 시작점도 폴 상단에서 시작하되, 곡선 제어점(cp)을 위쪽으로 올려 하늘 방향으로 자연스럽게 퍼지게 조정.
- 기대 결과: 폴대와 스트링이 건물 최상단 라인에 정확히 붙어 보임(더 이상 공중 부유 없음).

2) `StudioSidePanel.tsx` — 상단(빨강/파랑) 차량 Y축 8~10px 상향
- 현재 `top-[10%]`을 픽셀 기준으로 위로 이동(약 8~10px)하도록 조정.
- 상단 두 차량(🚗, 🚙) 모두 동일 기준으로 맞춰 차선 중앙(구분선 위) 정렬.

3) `StudioSidePanel.tsx` + `tailwind.config.ts`(필요 시) — 하단(택시/미니밴) 주행 안정화
- 조건 유지: 하단은 좌→우 이동, 차량 머리 방향은 진행 방향과 일치.
- 수정:
  - 택시/미니밴에 **고정 폭 wrapper**를 부여해 이모지 glyph 폭 차이 제거.
  - 애니메이션은 wrapper에, `scaleX(-1)`는 내부 이모지에만 적용(변형 충돌 방지).
  - 상/하단 차량 wrapper 폭을 통일해 “중간에서 생성/소멸” 현상 제거.
  - 필요하면 `car-move-right/left` 시작·종료 translate 범위를 wrapper 폭 기준으로 미세 조정.
- 기대 결과: 택시/미니밴도 다른 차량처럼 도로 양끝에서 자연 진입/이탈.

4) `StudioSidePanel.tsx` — 드럼을 기타보다 크게 + 더 높게
- 무대 악기 그룹에서 드럼 세트(킥/탐/심벌) 스케일 확대.
- 드럼 그룹 Y를 위로 올려 파라솔 캐노피보다 시각적으로 더 높게 보이도록 조정.
- 기타는 현재 크기 유지(또는 소폭 축소)로 상대 비율 명확화.

5) `src/components/worship-studio/StudioUnit.tsx` — 창문 밝기 통일(투명도 제거)
- 문제 원인: `windowGlow`가 알파 포함 그라데이션(`.../80`, `.../40`)이라 배경색 영향으로 층별 밝기가 달라 보임.
- 수정:
  - `windowGlow`를 불투명 단색(또는 불투명 2톤)으로 변경.
  - 아바타/이름/방문 버튼 창문 모두 동일 색상 토큰 사용.
- 기대 결과: 최상층~하층까지 창문 밝기가 동일하고 선명하게 유지.
