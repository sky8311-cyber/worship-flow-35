
문제 재정의
- 현재 모바일에서 빌딩 본체가 과도하게 커지고(빈 면적 증가), 1F/도로가 하단에서 잘리거나 가시성이 떨어집니다.
- 직접 원인: `ScrollArea`에만 `max-h-[40vh]`를 준 상태에서 상위 빌딩 body는 `flex-1`로 남아 있어, body 내부에 남는 빈 영역이 생깁니다.
- 부수 원인: 도로/차량 클리핑은 도로 컨테이너, 차량 y좌표, 인도 높이 조합이 일관되지 않은 상태입니다.

수정 계획 (StudioSidePanel.tsx 중심)
1) 모바일 레이아웃 복구 (기존 클리핑 방식 제거)
- `ScrollArea`의 `max-h-[40vh]` 제거.
- 대신 모바일에서 “빌딩 body 자체”를 제한:
  - body를 `h-[40~42vh] flex-none`로 두고, 데스크탑은 기존 `flex-1` 유지.
- 목표: body 내부 빈 공간 폭증을 막고, 1F/상가/도로를 항상 하단에 보이게 만듭니다.

2) 스크롤 책임 분리 고정
- body 내부 `ScrollArea`는 `h-full min-h-0`로 유지하여 층(ROOFTOP~1F)만 스크롤.
- `GroundFloorShops`/`AnimatedRoad`는 body 바깥 `shrink-0` 유지(고정 영역).
- 모바일/데스크탑 모두 같은 규칙으로 정리합니다.

3) 차량 비주얼 원복 + 누수 방지
- 빨강/파랑차 상단 차선 `top: -13px` 유지(원래 모양 복구).
- 인도 높이는 축소값(`h-3/h-4`) 유지해 가림 최소화.
- 도로 최상위 컨테이너 `overflow-hidden` 유지해 차량이 캔버스/페이지로 새지 않게 고정.

4) 모바일 시트 flex 안정화
- `mode="mobile"` 루트/랩퍼에 `min-h-0`를 명시해 높이 계산 충돌 방지.
- 필요 시 상단 spacer를 소폭 축소해 하단(상가/도로) 가시성 우선 확보.

검증 기준
- 모바일(430x659)에서 시트 오픈 직후 1F 라벨 + 상가 + 도로가 함께 보임.
- 층 스크롤 시 건물 층만 이동하고 상가/도로는 고정.
- 차량이 전체 페이지/캔버스로 넘어가지 않음.
- 빨강/파랑차가 인도에 과도하게 잘리지 않음.
- 데스크탑도 동일하게 “층만 스크롤, 하단 고정” 동작 유지.
