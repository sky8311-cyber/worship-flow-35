
## Fix iOS Native App Display Issues

### 1. Fix Status Bar Overlap (content hidden behind notch)

**Problem**: App content renders behind the iOS status bar because the viewport doesn't account for safe areas.

**Changes:**

**index.html** — Update the viewport meta tag:
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0, viewport-fit=cover" />
```

**capacitor.config.ts** — Add Status Bar and Splash Screen configuration:
```typescript
const config: CapacitorConfig = {
  appId: 'app.kworship.main',
  appName: 'K-Worship',
  webDir: 'dist',
  plugins: {
    StatusBar: {
      overlaysWebView: true,
      style: 'DARK'
    },
    SplashScreen: {
      launchAutoHide: true,
      autoHideDelay: 1500
    }
  }
};
```

**src/index.css** — Add safe-area padding to the root layout so content doesn't hide behind the status bar:
```css
body {
  padding-top: env(safe-area-inset-top, 0px);
}
```

**src/components/layout/AppLayout.tsx** — Add top safe-area class to the container:
```
<div className="min-h-[100dvh] bg-gradient-soft" style={{ paddingTop: 'env(safe-area-inset-top, 0px)' }}>
```

### 2. App Icon (manual step in Xcode)

The native iOS app icon must be set inside Xcode, not via the web favicon. This is a manual step:
1. Open Xcode project
2. Go to the **App** target -> **General** -> **App Icons and Launch Screen**
3. Open **Assets.xcassets** -> **AppIcon**
4. Drag your 1024x1024 icon into the slot -- Xcode auto-generates all required sizes

This cannot be done from Lovable code changes; it requires the Xcode Asset Catalog.

### 3. Splash Screen (manual step in Xcode)

The splash screen is configured via Xcode's **LaunchScreen.storyboard** (auto-generated by Capacitor). You can customize it in Xcode by editing that storyboard file, or keep the default white screen with the config above that auto-hides after 1.5 seconds.
